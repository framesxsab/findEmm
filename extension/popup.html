Before you start

Choose what findEmm may process.

When you click Capture, findEmm reads the visible name, title, company, and profile URL from the current LinkedIn profile or company page. Nothing is captured in the background, and you review every field.

When you click Research, those fields plus any work contact you entered go to the local API on this device. If its operator enabled Hunter Email Finder, the API sends either the LinkedIn profile handle or the person’s name and company domain to Hunter. Hunter is optional and off by default. A full name and company domain are required locally for every Hunter route so later opt-outs match both lookup forms.

If the operator explicitly enabled allowlisted company-page lookup, the local API may request only that approved company domain’s public contact page.

No data is sold, used for ads, or sent to a findEmm-operated cloud service.
