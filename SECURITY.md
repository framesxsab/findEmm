# Security policy

## Supported version

Security fixes are made on the latest version in the `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability or include sensitive details in a pull request.

Email `framessab@gmail.com` with:

- a clear description of the issue;
- steps to reproduce it safely;
- the affected version or commit; and
- any suggested mitigation, if available.

We will acknowledge the report, assess its impact, and coordinate a fix before public disclosure when appropriate.

## Handling user data

Do not include real contact details, vault data, pairing tokens, API keys, or suppression data in reports. Use synthetic examples wherever possible.
