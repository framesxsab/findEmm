# Publisher inputs required before submission

These values and decisions cannot be inferred safely and must be set by the publisher:

| Input | Required value |
| --- | --- |
| Public HTTPS privacy-policy URL | Host `privacy-policy.html` and paste its final URL. |
| Support email | Confirm `framessab@gmail.com` or replace it consistently in both privacy-policy files and Store fields. |
| Developer name | Legal or project name that will appear in Chrome Web Store. |
| Distribution | Public, unlisted, or trusted testers; choose intentionally. |
| Screenshots | Genuine captures following `SCREENSHOT_GUIDE.md`. |
| Hunter mode | Off by default. Enabling requires a user-owned key and retained evidence of Hunter's prior written commercial approval for this product use. |
| Company-page mode | Off by default. Enabling requires explicit operator approval and a reviewed exact-domain allowlist. |
| Recruiter CSV source | Use only recruiter-controlled exports containing work-contact data the operator is permitted to process. Map only explicit work/business headers and LinkedIn `/in/` person URLs; generic, personal, private, home, mobile, cell, and WhatsApp contact columns are rejected. |
| Manual release signoff | Clean-profile functional smoke plus rendered visual review of the exact packaged build. |

If GitHub Pages is chosen for the privacy policy, the repository owner must verify repository visibility and enable Pages. Do not claim the public URL exists until it resolves over HTTPS and serves the final policy.
